# 개발자를 위한 투두리스트 서비스 요구사항 정의서

## 1. 서비스 개요

### 1.1 서비스명

개발자를 위한 투두리스트 서비스

### 1.2 서비스 목적

개발자가 자신의 학습, 개발 업무, 프로젝트 작업을 효율적으로 관리할 수 있도록 할 일을 등록하고 관리하는 서비스이다.
또한 친구의 투두리스트를 참고하거나 커뮤니티 게시판을 통해 자유롭게 소통할 수 있도록 지원한다.

### 1.3 주요 기능

- 할 일 등록, 조회, 수정, 삭제
- 달력을 통한 날짜별 할 일 조회
- 이전 날짜의 할 일을 복사하여 현재 날짜에 붙여넣기
- 할 일 선택 후 타이머 실행
- 친구 추가 요청, 수락, 거절
- 친구의 투두리스트 조회
- 친구의 투두리스트는 조회만 가능하며 수정, 삭제 불가
- 커뮤니티 게시판을 통한 사용자 간 소통

# 8. 주요 사용자 시나리오

## 8.1 할 일 등록 시나리오

1. 사용자가 로그인한다.
2. 메인 화면으로 이동한다.
3. 할 일 제목과 내용을 입력한다.
4. 날짜, 카테고리, 우선순위를 선택한다.
5. 등록 버튼을 클릭한다.
6. 할 일 목록에 새 할 일이 표시된다.

---

## 8.2 이전 날짜 할 일 복사 시나리오

1. 사용자가 할 일 등록 화면에서 이전 날짜를 선택한다.
2. 해당 날짜의 할 일 목록이 표시된다.
3. 사용자가 복사할 할 일을 선택한다.
4. 복사 버튼을 클릭한다.
5. 현재 날짜의 할 일 등록 영역에 내용이 입력된다.
6. 사용자가 저장 버튼을 클릭한다.
7. 현재 날짜에 복사된 할 일이 등록된다.

---

## 8.3 친구 추가 시나리오

1. 사용자가 친구 화면으로 이동한다.
2. 닉네임 또는 이메일로 사용자를 검색한다.
3. 친구 요청 버튼을 클릭한다.
4. 상대방의 친구 요청 목록에 요청이 표시된다.
5. 상대방이 수락하면 친구 관계가 성립된다.
6. 거절하면 친구 관계가 생성되지 않는다.

---

## 8.4 친구 투두리스트 조회 시나리오

1. 사용자가 친구 목록 화면으로 이동한다.
2. 친구 투두 보기 버튼을 클릭한다.
3. 친구의 투두리스트 화면으로 이동한다.
4. 날짜별로 친구의 할 일을 조회한다.
5. 수정, 삭제, 타이머 버튼은 제공되지 않는다.

---

## 8.5 타이머 사용 시나리오

1. 사용자가 자신의 할 일 목록에서 하나의 할 일을 선택한다.
2. 타이머 시작 버튼을 클릭한다.
3. 타이머가 실행된다.
4. 사용자가 일시정지 또는 종료 버튼을 클릭한다.
5. 종료 시 누적 시간이 해당 할 일에 저장된다.

---

## 8.6 커뮤니티 게시판 사용 시나리오

1. 사용자가 커뮤니티 화면으로 이동한다.
2. 게시글 목록을 조회한다.
3. 글쓰기 버튼을 클릭한다.
4. 제목, 내용, 카테고리를 입력한다.
5. 등록 버튼을 클릭한다.
6. 게시글 목록에 새 게시글이 표시된다.
7. 다른 사용자는 게시글을 조회하고 댓글을 작성할 수 있다.
