# 개발자를 위한 투두리스트 서비스 테이블 설계서

## 1. 문서 개요

이 문서는 개발자를 위한 투두리스트 서비스의 데이터베이스 테이블 설계서이다.

> DBMS는 Oracle을 기준으로 작성

---

## 2. 테이블 목록

| 테이블명        | 설명                    |
| --------------- | ----------------------- |
| users           | 사용자 정보             |
| todos           | 할 일 정보              |
| todo_categories | 사용자별 할 일 카테고리 |
| friends         | 친구 요청 및 친구 관계  |
| posts           | 커뮤니티 게시글         |
| comments        | 게시글 댓글             |

---

# 3. users 테이블

사용자 정보를 저장하는 테이블이다.

| 컬럼명      | 타입          | 제약조건                             | 설명                           | 값 예시             |
| ----------- | ------------- | ------------------------------------ | ------------------------------ | ------------------- |
| user_id     | NUMBER(19)    | PK, GENERATED BY DEFAULT AS IDENTITY | 사용자 ID                      | 1                   |
| email       | VARCHAR2(100) | NOT NULL, UNIQUE                     | 사용자 이메일                  | user@example.com    |
| nickname    | VARCHAR2(50)  | NOT NULL, UNIQUE                     | 닉네임                         | 홍길동              |
| provider    | VARCHAR2(20)  | NOT NULL                             | 소셜 로그인 제공자             | KAKAO, GOOGLE       |
| provider_id | VARCHAR2(100) | NOT NULL                             | 제공자가 부여한 사용자 고유 ID | 1234567890          |
| status      | VARCHAR2(20)  | NOT NULL                             | 계정 상태                      | ACTIVE, DELETED     |
| created_at  | TIMESTAMP     | NOT NULL                             | 가입일                         | 2026-05-22 10:00:00 |
| updated_at  | TIMESTAMP     | NULL                                 | 수정일                         | 2026-05-22 10:30:00 |
| deleted_at  | TIMESTAMP     | NULL                                 | 탈퇴일                         | 2026-05-22 11:00:00 |

## 3.1 users 상태값

| 상태값  | 설명        |
| ------- | ----------- |
| ACTIVE  | 정상 계정   |
| DELETED | 탈퇴한 계정 |

# 6. friends 테이블

친구 요청 및 친구 관계를 저장하는 테이블이다.

| 컬럼명       | 타입         | 제약조건                             | 설명                       | 값 예시                              |
| ------------ | ------------ | ------------------------------------ | -------------------------- | ------------------------------------ |
| friend_id    | NUMBER(19)   | PK, GENERATED BY DEFAULT AS IDENTITY | 친구 관계 ID               | 1                                    |
| requester_id | NUMBER(19)   | FK, NOT NULL                         | 친구 요청을 보낸 사용자 ID | 1                                    |
| receiver_id  | NUMBER(19)   | FK, NOT NULL                         | 친구 요청을 받은 사용자 ID | 2                                    |
| status       | VARCHAR2(20) | NOT NULL                             | 친구 요청 상태             | PENDING, ACCEPTED, REJECTED, DELETED |
| requested_at | TIMESTAMP    | NOT NULL                             | 요청일                     | 2026-05-22 10:00:00                  |
| responded_at | TIMESTAMP    | NULL                                 | 수락 또는 거절일           | 2026-05-22 10:30:00                  |
| deleted_at   | TIMESTAMP    | NULL                                 | 친구 삭제일                | 2026-05-22 11:00:00                  |

## 6.1 friends 상태값

| 상태값   | 설명              |
| -------- | ----------------- |
| PENDING  | 친구 요청 대기 중 |
| ACCEPTED | 친구 요청 수락    |
| REJECTED | 친구 요청 거절    |
| DELETED  | 친구 관계 삭제    |

## 6.2 friends 제약조건

| 제약조건                           | 설명                          |
| ---------------------------------- | ----------------------------- |
| CHECK(requester_id <> receiver_id) | 자기 자신에게 친구 요청 방지  |
| UNIQUE(requester_id, receiver_id)  | 동일 방향 중복 친구 요청 방지 |

# 9. 테이블 관계 요약

| 관계                        | 설명                                                   |
| --------------------------- | ------------------------------------------------------ |
| users 1 : N todos           | 한 명의 사용자는 여러 할 일을 작성할 수 있다.          |
| users 1 : N todo_categories | 한 명의 사용자는 여러 할 일 카테고리를 생성할 수 있다. |
| todo_categories 1 : N todos | 하나의 카테고리는 여러 할 일에 연결될 수 있다.         |
| users 1 : N posts           | 한 명의 사용자는 여러 게시글을 작성할 수 있다.         |
| users 1 : N comments        | 한 명의 사용자는 여러 댓글을 작성할 수 있다.           |
| posts 1 : N comments        | 하나의 게시글에는 여러 댓글이 작성될 수 있다.          |
| users N : N users           | friends 테이블을 통해 사용자 간 친구 관계를 관리한다.  |

# 10. 소프트 삭제 기준

이 설계에서는 데이터를 실제로 삭제하지 않고 `status`와 `deleted_at`을 이용해 소프트 삭제를 처리한다.

| 컬럼       | 역할                             |
| ---------- | -------------------------------- |
| status     | 데이터의 현재 상태를 나타낸다.   |
| deleted_at | 삭제된 시간을 기록한다.          |
| created_at | 데이터가 생성된 시간을 기록한다. |
| updated_at | 데이터가 수정된 시간을 기록한다. |

일반 조회 시에는 삭제된 데이터를 제외하기 위해 다음 조건을 사용한다.

```sql
WHERE deleted_at IS NULL
```

또는 상태값을 함께 사용할 수 있다.

```sql
WHERE status = 'ACTIVE'
  AND deleted_at IS NULL
```

단, `todos.status`는 할 일 진행 상태를 의미하므로 삭제 여부는 `deleted_at`으로 판단하는 것이 좋다.
